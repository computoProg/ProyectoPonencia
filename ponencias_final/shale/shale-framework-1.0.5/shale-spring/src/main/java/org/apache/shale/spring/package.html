The package contains an optional integration layer connecting Shale with the Spring Framework. The following capabilities are enabled by this integration:

  • Spring's DelegatingVariableResolver is automatically enabled, which will utilize Spring's standard facilities to create a new bean if the standard JavaServer Faces managed bean facility does not recognize this variable name.
  • An additional resolver is added that maps the (configurable) variable name webApplicationContext to the instance of org.springframework.web.context.WebApplicationContext that Spring has established for this web application.

To enable this integration in applications based on Shale, simply include the following JAR files in your application's /WEB-INF/lib directory, in addition to shale.jar:

  • From the Shale Core Library distribution: shale-spring.jar.
  • From the Spring Framework distribution (version 1.1.5 or later): spring-context.jar, spring-core.jar, and spring-web.jar (alternatively, include spring.jar which includes all of the above)
